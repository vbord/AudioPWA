AB Product Roadmap

ver_AB_prd_01 → ver_AB_prd_04 — baseline, local, video, upload

ver_AB_prd_01 — current stable baseline

  • Tree works stably.
  • Custom player with drag, snapping, haptics.
  • Search, jump, resume, API all wired.
  • UI is clean, compact, mobile-friendly.
  • Code is starting to be modular (tree-search.js, custom-player.js).

This is the right moment to fix this version on GitHub as a safe baseline.

ver_AB_prd_02 — play folders from local device

Goals

  • Access local filesystem via File System Access API.
  • Let user pick a folder (showDirectoryPicker()).
  • Recursively walk files and build the tree locally (no server API).
  • Unify UI: local and server books look identical in the tree.
  • Optionally cache covers and descriptions if present.

Why it fits the current architecture

  • Tree already works with arbitrary data sources.
  • Player is independent of API.
  • Progress can be keyed by file path → works for local files too.

ver_AB_prd_03 — add video playback (local + server)

Goals

  • Support video formats (e.g. .mp4, .mkv, .mov).
  • Add a dedicated <video> element.
  • Switch between audio and video modes in the same UI.
  • Optionally show thumbnails or previews.
  • Reuse existing logic: drag, snapping, resume, tree navigation.

Most of the work is already done — video becomes just another file type.

ver_AB_prd_04 — upload (local → server)

Goals

  • Support drag-and-drop or file/folder selection for upload.
  • Upload via API with progress indication.
  • Automatically create server-side structure from local folders.
  • Refresh tree after upload to reflect new content.

Why this step later

  • By then, UI for local + server playback is already unified.
  • Tree and player are generic and stable.
  • Server side can stay minimal and focused on storage + metadata.

Overall strategy

  1. Stable base — ver_AB_prd_01 (done).
  2. Local folders — ver_AB_prd_02 (autonomous mode).
  3. Video support — ver_AB_prd_03 (feature expansion).
  4. Upload — ver_AB_prd_04 (full platform).

This order is logical: each step builds on the previous one without breaking your workflow.

Immediate next step

  • Commit and push ver_AB_prd_01 to GitHub as the baseline.
  • Create a branch like ver_AB_prd_02_dev for local-folder work.
  • Optionally extract 1–2 more modules (e.g. progress, api) for cleaner structure.
